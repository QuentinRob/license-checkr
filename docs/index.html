Policy engine

Your rules.
Your project.

Drop a license-checkr.toml at your project root and define exactly which licenses are acceptable, which need review, and which are blocked.

No config file? Built-in defaults are already sensible â€” permissive licenses pass, weak-copyleft warn, and GPL/AGPL are errors.

  • Per-license rules keyed by SPDX identifier
  • A catch-all default for anything unlisted
  • Project-level or global ~/.config config
  • Override with --config <path> at runtime
license-checkr.toml
# Verdict for any license not listed below
[policy]
default = "warn"

[policy.licenses]
# Permissive â€” always allowed
"MIT"          = "pass"
"Apache-2.0"   = "pass"
"BSD-3-Clause" = "pass"
"ISC"          = "pass"
"Unlicense"    = "pass"

# Weak copyleft â€” review required
"LGPL-2.1"     = "warn"
"MPL-2.0"      = "warn"

# Strong copyleft â€” blocked
"GPL-3.0"      = "error"
"AGPL-3.0"     = "error"

How it works

Three steps to compliance

1

Install

A single binary, no runtime dependencies. Install globally or add to your CI environment in seconds.

cargo install license-checkr
2

Scan

Run at the root of any project. Ecosystems are detected automatically. Add --online to resolve missing licenses from registries.

license-checkr --online
3

Enforce

Add to your CI pipeline. Exits 1 on policy violations so you can block merges before problematic licenses sneak in.

license-checkr -q
